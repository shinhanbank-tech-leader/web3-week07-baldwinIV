## 핵심 질문 (참고용)

아래 질문을 참고하여 작성하면 도움이 됩니다.

- 스마트컨트랙트는 왜 “업그레이드가 어려운 구조”인가  
- delegateCall 기반 업그레이드 방식은 어떤 원리로 동작하는가  
- Proxy 패턴(로직/데이터 분리)은 어떤 문제를 해결하기 위해 등장했는가  
- 은행 시스템에서의 배포/변경 방식과 무엇이 다른가  
- 스마트컨트랙트 업그레이드 방식에서 발생할 수 있는 위험은 무엇인가  
- 은행 시스템에서는 변경 시 어떤 통제(승인, 검증, 롤백 등)를 수행하는가  
- 두 구조를 비교했을 때 운영·통제 관점에서 가장 큰 차이는 무엇인가  

---

## 작성 가이드

- 기술 개념 설명만 나열하지 말고,  
  **은행 IT / 운영 / 통제 관점에서의 의미**를 함께 작성해주세요  

- “왜 이런 구조가 나왔는지” + “운영 시 어떤 문제가 생기는지” 중심으로 작성  

- 가능하면 실제 은행 시스템 예시와 연결  
  (예: 코어 변경, 배치 수정, 승인 프로세스 등)
